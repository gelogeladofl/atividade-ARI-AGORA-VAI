<h2>⚙️ Funcionamento do projeto ⚙️</h2>
<li>MÉTODO GET:</li>
Visualizar uma página web.

Realizar uma pesquisa (o Google usa GET).

Solicitar informações detalhadas de um recurso, como obter dados de um voo ou produto.

Recupera dados, visível na URL, seguro (não altera estado)., limitado no tamanho.

Não alteramos o servidor.
<li>MÉTODO POST:</li>
Dados no corpo: Diferente do GET, os dados não aparecem na URL, oferecendo maior privacidade.

Sem limite de tamanho: Não possui a restrição de caracteres da URL, permitindo envio de grandes volumes de dados.

Não idempotente: Múltiplas requisições idênticas podem criar vários recursos no servidor.

Uso comum Envio de formulários, cadastros e uploads de arquivos.

Cria/altera dados, invisíveis na URL, inseguro (altera estado).
